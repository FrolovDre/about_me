# Портфолио‑сайт соискателя

Этот репозиторий содержит исходный код для современного портфолио‑сайта, разработанного на базе **Next.js (App Router)** c использованием **TypeScript** и **Tailwind CSS**. Проект полностью готов к сборке и деплою — замените демо‑данные на свои и опубликуйте сайт на Vercel или GitHub Pages.

## Быстрый старт

1. **Установка зависимостей:**

   ```bash
   npm install
   # или
   yarn install
   ```

2. **Запуск разработки:**

   ```bash
   npm run dev
   # приложение станет доступно на http://localhost:3000
   ```

3. **Сборка в production:**

   ```bash
   npm run build
   npm start
   ```

4. **Настройка контента:**

   Вся информация о вас хранится в файле `data/profile.ts`. Здесь можно заменить имя, контактные данные, краткое описание, навыки, список проектов, опыт работы и образование. Изображения помещайте в папку `public/` и обновляйте соответствующие поля в объектах данных.

5. **Режимы light/dark:**

   Компонент переключения темы сохраняет выбор в `localStorage`. При первом посещении тема определяется системными настройками пользователя.

## Деплой

### Вариант 1: Vercel (рекомендовано)

1. Авторизуйтесь на [vercel.com](https://vercel.com/) и создайте новый проект, указав репозиторий.
2. В настройках проекта достаточно выбрать фреймворк **Next.js** — остальные параметры будут определены автоматически.
3. После деплоя вы получите собственный URL вида `https://username.vercel.app`. При коммитах в основную ветку Vercel будет автоматически пересобирать сайт.

### Вариант 2: GitHub Pages

Next.js требует Node.js сервер во время работы, поэтому для статического хостинга на GitHub Pages можно воспользоваться инструментом `next export`, который превращает приложение в набор статических файлов. Однако некоторые функции (динамические маршруты) будут недоступны. Альтернативой является использование **Astro** или **Vite** для полностью статических сайтов. В этом репозитории реализован Next.js — для GitHub Pages выполните:

```bash
npm run build
npm run export
```

Все файлы появятся в каталоге `out/`. Затем включите GitHub Pages в настройках репозитория и укажите папку `out` в качестве источника. Учтите, что формы на странице контактов при таком варианте необходимо отправлять через сервисы вроде [Formspree](https://formspree.io/), либо заменить на `mailto:` ссылку.

## Структура проекта

- **`app/`** – страницы и layout; корневая страница (`page.tsx`) содержит секции портфолио.
- **`components/`** – переиспользуемые компоненты: навигация, секции, карточки проектов и т.д.
- **`data/`** – файл `profile.ts` с контентом сайта.
- **`public/`** – изображения, favicon и заглушки (например, `CV.pdf`).
- **`styles/`** – глобальные стили (подключаются в `app/layout.tsx`). TailwindCSS отвечает за основную стилизацию.
- **`.gitignore`** – исключает из репозитория временные файлы, сборки и зависимости.
- **`.eslintrc.json` / `prettier.config.js`** – конфигурации линтера и форматтера для единого стиля кода.

## SEO

В компоненте `Head` каждой страницы определяются базовые мета‑теги (`<title>`, `<meta description>`, Open Graph). Для социальных сетей можно заменить `public/og.png` на своё изображение (1200×630 px) и обновить мета‑тег `og:image` в `layout.tsx`.

## Контакты и форма

Кнопка «Скачать резюме» ссылается на файл `public/CV.pdf`. В форме обратной связи используется атрибут `mailto:` — письма будут открываться в клиенте пользователя. При желании можно подключить внешний сервис (например, Formspree), для этого в компоненте `ContactForm` следуйте инструкции в комментариях.

---

Сайт полностью адаптивный и доступный: применяются семантические теги, состояния фокуса для интерактивных элементов, тёмная и светлая темы. Демо‑контент легко заменить — отредактируйте `data/profile.ts` и изображения в `public/`.
